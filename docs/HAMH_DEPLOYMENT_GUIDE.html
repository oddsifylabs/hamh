0Prerequisites

Before you begin, ensure you have the following:

  • Hostinger VPS account with SSH access
  • SSH key pair for connecting to Miah Hermes VPS
  • A domain name (for SSL certificate)
  • Basic Linux/bash command line knowledge
  • Text editor knowledge (nano, vi, etc.)

1Initial VPS Setup

1.1 SSH into your Hostinger VPS

ssh root@your-hostinger-ip-address

Replace your-hostinger-ip-address with your actual Hostinger VPS IP.

1.2 Update system packages

apt update && apt upgrade -y

This ensures all system packages are up-to-date and secure.

1.3 Install Node.js and npm

curl -sL https://deb.nodesource.com/setup_18.x | sudo -E bash -
apt install -y nodejs
node --version  # Should be v18+
npm --version

This installs Node.js v18 (or later) and npm, required for running HAMH.

1.4 Install additional tools

apt install -y git curl wget nano supervisor
✓ Step 1 Complete: Your VPS is now prepared with Node.js and essential tools.

2Clone/Setup Manager Agent

2.1 Create app directory

mkdir -p /app/manager-agent
cd /app/manager-agent

2.2 Copy files to VPS

Option A: Using git (recommended)

git clone https://github.com/oddsify-labs/hamh.git .
cd /app/manager-agent

Option B: Using scp from your local machine

scp package.json root@your-hostinger-ip:/app/manager-agent/
scp manager-agent-server.js root@your-hostinger-ip:/app/manager-agent/
scp .env.example root@your-hostinger-ip:/app/manager-agent/

2.3 Install dependencies

cd /app/manager-agent
npm install

2.4 Create and configure .env file

cp .env.example .env
nano .env

Edit the following variables in your .env file:

PORT=3000
NODE_ENV=production

# Miah Hermes (VPS SSH)
MIAH_HOST=your-miah-vps-ip
MIAH_USER=root
MIAH_KEY_PATH=/root/.ssh/miah_key

# Markus Bot (Local HTTP)
MARKUS_API=http://192.168.1.100:3001

# Alexbet Sharp (Railway HTTPS)
ALEXBET_API=https://alexbet-sharp.railway.app
ALEXBET_KEY=your-railway-api-key

Save the file with Ctrl+X, then Y, then Enter.

✓ Step 2 Complete: HAMH is now installed and configured.

3Setup SSH Keys (for Miah Hermes)

3.1 Create SSH directory

mkdir -p /root/.ssh
chmod 700 /root/.ssh

3.2 Add Miah Hermes SSH key

Copy your private SSH key from your local machine:

# From your local machine:
scp ~/.ssh/miah_key root@your-hostinger-ip:/root/.ssh/miah_key

Set proper permissions:

chmod 600 /root/.ssh/miah_key

3.3 Test SSH connection

ssh -i /root/.ssh/miah_key root@miah-vps-ip "echo 'SSH working!'"
✓ Step 3 Complete: HAMH can now connect to Miah Hermes via SSH.

4Run Manager Agent

4.1 Test locally first

cd /app/manager-agent
npm start

You should see:

╔════════════════════════════════════════╗
║  MANAGER AGENT SERVER                  ║
║  Oddsify Labs - CEO Command Center    ║
║  Running on port 3000                  ║
╚════════════════════════════════════════╝

Test the health endpoint in another terminal:

curl http://localhost:3000/health

Should return:

{"status":"healthy","timestamp":"...","uptime":...}

Stop the server with Ctrl+C.

4.2 Setup PM2 (production process manager)

npm install -g pm2
pm2 start manager-agent-server.js --name "manager-agent"
pm2 save
pm2 startup

Verify it's running:

pm2 status
✓ Step 4 Complete: HAMH is now running with PM2 in production mode.

5Setup Reverse Proxy (Nginx)

5.1 Install Nginx

apt install -y nginx

5.2 Create Nginx configuration

nano /etc/nginx/sites-available/manager-agent

Paste the following configuration:

server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # CORS headers
    add_header 'Access-Control-Allow-Origin' '*' always;
    add_header 'Access-Control-Allow-Methods' 'GET, POST, PUT, DELETE, OPTIONS' always;
    add_header 'Access-Control-Allow-Headers' 'Content-Type, Authorization' always;

    if ($request_method = 'OPTIONS') {
        return 204;
    }
}

Replace your-domain.com with your actual domain.

5.3 Enable the configuration

ln -s /etc/nginx/sites-available/manager-agent /etc/nginx/sites-enabled/
rm /etc/nginx/sites-enabled/default
nginx -t
systemctl restart nginx
✓ Step 5 Complete: Nginx is now proxying requests to HAMH on port 3000.

6SSL Certificate (Let's Encrypt)

6.1 Install Certbot

apt install -y certbot python3-certbot-nginx

6.2 Get SSL certificate

certbot --nginx -d your-domain.com

Follow the interactive prompts. Certbot will automatically update your Nginx configuration.

6.3 Enable auto-renewal

systemctl enable certbot.timer
systemctl start certbot.timer
✓ Step 6 Complete: Your domain now has an SSL certificate with auto-renewal enabled.

7Firewall Setup

7.1 Enable UFW (Uncomplicated Firewall)

ufw allow 22/tcp   # SSH
ufw allow 80/tcp   # HTTP
ufw allow 443/tcp  # HTTPS
ufw enable
✓ Step 7 Complete: Firewall is configured to allow HTTP, HTTPS, and SSH.

8Connect Dashboard

In your HAMH dashboard HTML, add the following script tags:

<script>
  window.MANAGER_AGENT_URL = 'https://your-domain.com';
</script>
<script src="dashboard-api-client.js"></script>

Your dashboard will now connect to your deployed HAMH instance.

10Troubleshooting

Manager Agent won't start

pm2 logs manager-agent --err

Check the error output and verify your .env configuration.

Can't connect to Miah Hermes

ssh -i /root/.ssh/miah_key root@miah-vps-ip

Verify the SSH key is present and correct IP address is configured.

Nginx 502 Bad Gateway

Check the following:

  • Node.js is running: pm2 status
  • Firewall allows traffic: ufw status
  • Nginx config is valid: nginx -t

CORS errors in dashboard

Ensure your Nginx config includes:

add_header 'Access-Control-Allow-Origin' '*' always;

11Security Checklist

Before going to production, verify:

  • SSH keys protected with correct permissions (chmod 600)
  • Firewall enabled (UFW) with only necessary ports open
  • SSL certificate installed and auto-renewal configured
  • Environment variables in .env (never hardcoded)
  • API keys never logged to console
  • Database credentials secured
  • Regular backups enabled
  • PM2 startup script configured for automatic restart on reboot

12Scaling (Future)

When you're ready to scale beyond a single VPS:

  1. Database: Add MongoDB or PostgreSQL for persistent task storage
  2. Cache Layer: Add Redis for task queue caching and performance
  3. Load Balancing: Deploy multiple HAMH instances behind a load balancer
  4. Monitoring: Add DataDog, New Relic, or similar monitoring solution
  5. Backup Strategy: Implement automated daily backups to cloud storage
